Advanced settings tab

This tab allows the use of advanced settings. The options are different according to the emulator being used.

Processor

It may be necessary to emulate a different processor depending on your ROM image. The processor types supported by Basilisk II are Motorola 68020, 68030 and 68040. For SheepShaver virtual machines, the processor is always PowerPC 7400 (G4).

Floating-point instructions

For Basilisk II machines, there is an option for floating-point instructions (FPU) emulation. This enables the emulation of the co-processor 68881 and, in case 68040 is selected, the FPU is emulated within the processor emulation. Sometimes the ROM image requires the FPU setting. Pay attention to the information available about the ROM Image at the Assets Manager.

JIT compilation

A Just-In-Time (JIT) translation engine aims at translating 68040 instructions to native equivalent code sequences, thus providing faster emulation speeds.

Set this to "true" to enable the JIT compiler in Basilisk II and SheepShaver. The default value is "true". This feature is enabled only if the virtual machine is configured to emulate a 68040 processor.

Lazy cache invalidation

Set this to "true" to enable lazy invalidation of the translation cache on Basilisk II. This is always recommended as it usually makes the system more responsive and faster, especially while running MacOS 8.X. Default value is "true".

JIT cache size

This field allows to specify size in kilobytes of RAM for the translation cache. The value given will be rounded down to the nearest multiple of a page size. Minimal value is "2048" (2MB). Default value is "8192" (8MB). This field is visible only for Basilisk II machines.

68K dynamic recompilation

This checkbox enables 68K dynamic recompilation emulator on SheepShaver.

Do not convert clipboard contents

Don't convert clipboard contents. Default value is "false".

Sleep when idle

When this option is enabled, the virtual machine will sleep when the emulator becomes idle.

Hidden settings

There is a hidden option to use processors 68000 and 68010 and 68060. The 68000 processor was used in a few Mac models and 68010 processor was used nowhere. Basilisk II can, however, emulate these processors. This feature was not properly tested and may result in crashes.

To learn how to enable hidden features, refer to the preferences help page.
