Virtual Machines Window

This is the main window. This window is responsible for displaying all virtual machines you have and give you tools to do use and configure them as well as to delete, duplicate or create new virtual machines.

The layout is simple: there is a main list and an area with buttons.

The main list

The main list shows different virtual machines per row. Each virtual machine can run independently, but can also share characteristics of references to files used by other virtual machines.

There are three elements in the list: an icon, a name and a power button. The names cannot be duplicated. The buttons runs the virtual machine or, if it is already running, it will ask the emulator to stop. In Basilisk II, it means the guest operating system will prompt you to shut down, whereas in SheepShaver it will simply turn off.

The icon reflects the state of the virtual machine of that row. There are 4 possible icons:

The 'new file' icon

Means that this virtual machine has no relevant settings yet. It has no ROM file nor disk images associated with it. without these elements, it cannot be used.

The 'floppy question mark' icon

It means this virtual machine can not boot because there are no disks associated with it. When it is black and white, it means it is using an old world ROM. When it is coloured, it means it is using a new world ROM.

The 'broken mac' icon

It means the virtual machine can not boot because there is no ROM image associated with it.

The 'happy mac' icon

It means Medusa has all information necessary to get this virtual machine running. When it is black and white, it means it is using an old world ROM. When it is coloured, it means it is using a new world ROM.

When the virtual machine is being used, a picture of a lock will be displayed over the icon. It means the virtual machine cannot be modified, deleted or duplicated. Also, all of the disks being used are being used in read-write mode, which means that a parallel emulation could corrupt the file. Because of that, Medusa will block any attempts to execute a parallel virtual machine that uses the same disk images.

The buttons area

The buttons area encompasses 5 buttons which will trigger different actions:

The 'add' button

Allows the creation of a new virtual machine. Only the name of the virtual machine is necessary. All other settings, except the file associations will be set automatically.

The 'delete' button

Allows a selected virtual machine to be excluded. The preferences file, if generated, will be deleted and the virtual machine will be deleted. The associated files will remain untouched. There is no turning back from this action.

The 'duplicate' button

Allows the duplication of a selected virtual machine. If no machine is selected in the list, it becomes disabled. The second virtual machine will be a copy of the selected one. The name must be different.

The 'settings' button

This button with a gear wheel icon opens another window (virtual machine window) to change the selected virtual machine's settings.

The 'assets manager' button

This is a shortcut to open the Assets Manager.

Extra buttons

There is a preference to activate extra buttons in this area. You can learn more details in the preferences' help page.
