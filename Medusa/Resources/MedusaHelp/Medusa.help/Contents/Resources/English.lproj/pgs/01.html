About Medusa

Medusa is a modern emulation settings editor that makes it easy to setup several "virtual machines" to be used with the same emulator. Medusa lets you skip the line by line editing of the preferences file of the Basilisk II emulator that is normally required to run a single virtual machine. All related files can be easily organised and used by different collections of preferences called "virtual machines" by choosing the best fitting options for your emulation environment. You can use the same files with different settings and no invisible preferences files will be stored in your home folder.

Learning the basics

In Medusa Basics you'll learn how Basilisk II works. You will also learn what files you must have to use it, and how to load these files into Medusa's environment.

Some advanced tips

In this section you'll learn about some of the more powerful features of Medusa. There are some very flexible and powerful actions that come with Medusa. For example, you can add a UNIX command, shell script, or AppleScript script to an action and enable your workflow to do anything that a script can.

You can expand Medusa by adding new actions or completed workflows to your library. More actions are being created all the time by Apple and other software developers. If you are a scripter or programmer you can create your own actions.

Medusa is also a scriptable application, so you can use AppleScript scripts to control it.
