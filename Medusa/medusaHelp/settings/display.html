Display settings tab

The display settings tab allows you to choose a resolution, colour depth and frame update for the emulation.

Width and height

These fields allow you to define a size, in pixels, for the width and height of the emulator's window. This size is the actual virtual machine's display's size and the guest system's desktop size, ignoring the size of the title bar.

Fullscreen

Tick the "use fullscreen" checkbox to have fullscreen enabled. If this option is active, the emulator will ignore the width and height values from the previous fields and will execute in full screen.

8 15 24

Colour depth

Colour depth is the number of bits used for each colour component of a single pixel. Basilisk II and SheepShaver support 8, 15 and 24 bits of colour depth.

8 bits are equivalent to 256 colours, 15 bits to 32,768 colours (4,096 colours with 16 levels of transparency) and 24 bits are equivalent to 16,777,216 colour variations, also known as "true colour".

The number of colours the guest system uses depends also of its own settings, version and the ROM file. If you don't know which option is better, try different options.

Frame skipping

This setting specifies how many frames to skip after drawing one frame. Higher values make the video display more responsive but require more processing power. A value of "0" selects a "dynamic" update mode that cuts the display into rectangles and updates each rectangle individually, depending on display changes. The default value in Medusa is "0". Basilisk II and SheepShaver original default is "8".

QuickDraw acceleration

This option is available to SheepShaver machines only. It turns on QuickDraw acceleration. Default value in Medusa is "false", whereas in SheepShaver the default value is "true".
